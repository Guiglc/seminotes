## 3. 入门建议

1. **先通读 main.cpp 和 main.h**，理解主流程和全局变量。
2. **看 flow.cpp**，了解测试流程是如何组织和调用各测试项的。
3. **逐步阅读 testitem.cpp**，每个测试函数就是一个测试用例，配合 flow.cpp 理解调用关系。
4. **查阅 dc.cpp/device.cpp/level.cpp 等**，理解每个测试项内部调用的底层操作。

## 5. Note

你的项目中，`main` 可执行文件的名字和位置是由 `makefile.source` 里的这一行决定的：

```makefile
PRO=main	#The main program name after compiling
```

这表示：
- 编译后生成的主程序文件名就是 `main`，位于当前目录（SRAM_RP）。
- 如果你想让它生成在 `bin` 目录下，可以把这行改成 `PRO=bin/main`。
